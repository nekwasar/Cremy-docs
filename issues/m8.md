# M8: Document Generation Features

## Overview

**Development Approach:** Backend First
Document generation features including text input, AI editing, formatting, translation, and summarization. NO STYLING - pure functionality.

**Development Notes:**
- No Tailwind CSS - not needed
- No custom UI components with styling - use raw HTML/CSS only when required for testing
- Focus on: Generation works, editing works, formatting works, translation works, summarization works
- Add styling ONLY when user specifically requests it

**Tech Stack:**
- Frontend: React components + API routes
- AI: DeepSeek API (via M4)
- State: Zustand (from M3)
- WebSocket: Socket.io for streaming

**Milestone Dependencies:** M1 (Infrastructure), M2 (Frontend Core), M3 (State Management), M4 (Generation Engine), M5 (Generate Page), M7 (Preview)

## Issue 001: Generation from Text Input

### Description
Implement document generation from direct text input.

### Must Have
- Input methods:
  1. Direct Text: User types in main input box
  2. Template Selection: User selects format, then inputs text
  3. AI Auto-Detect: User enters text, AI determines format
- Processing flow:
  - User input (text)
  - Count words → Calculate credits required
  - Check user has enough credits
  - Send to AI with format context (if selected)
  - WebSocket streams response
  - Render in preview
  - Deduct credits
- Word count: 1 credit per 100 words

### Dependencies
- Milestone: M4 (AI Generation)
- Milestone: M5 (Generate Page)
- Milestone: M1_4 (Credit System)

### Files
- app/api/generate/route.ts
- src/lib/generate-from-text.ts

### Acceptance Criteria
- [ ] Generation from text works
- [ ] Credit calculation correct
- [ ] Streaming works
