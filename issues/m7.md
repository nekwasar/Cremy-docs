# M7: Document Preview & Rendering

## Overview

**Development Approach:** Backend First
Document preview and rendering system with view/edit modes, component rendering, tool-specific previews, and visual indicators. NO STYLING - pure functionality for document display and editing.

**Development Notes:**
- No Tailwind CSS - not needed
- No custom UI components with styling - use raw HTML/CSS only when required for testing
- Focus on: Document renders, edit modes work, previews work per tool
- Add styling ONLY when user specifically requests it

**Tech Stack:**
- Frontend: React components
- State: Zustand (from M3)
- Real-time: Server-sent events for streaming

**Milestone Dependencies:** M1 (Infrastructure), M2 (Frontend Core), M3 (State Management), M4 (Generation Engine), M5 (Generate Page), M19 (Agent)

---

## The 11 Document Tools

| # | Tool | Preview Behavior |
|---|------|-----------------|
| 1 | Generate (Text/Command) | Skeleton → first page preview → download + preview buttons |
| 2 | Edit | Done in preview page |
| 3 | Convert | Before/after preview (first page), 100% quality message |
| 4 | Translate | Before/after preview (both docs), download + preview |
| 5 | Voice | First page preview, download + preview |
| 6 | Extract (OCR) | Text ONLY, NO preview, copy + try again buttons |
| 7 | Merge | Ask order first, preview + download |
| 8 | Split | Ask pages to split, preview + download |
| 9 | Compress | Two PDF icons + sizes, download + preview |
| 10 | Change Style | After style selected → convert to text → recreate |
| 11 | Visual Indicators | Replace upload dash on agent pages |
