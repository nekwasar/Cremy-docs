# M4: Document Generation Engine

## Overview

**Development Approach:** Backend First
AI-powered document generation engine - middleware that connects user input to AI API, handles WebSocket streaming, and produces structured document output. NO STYLING - pure functionality.

**Development Notes:**
- No Tailwind CSS - not needed
- No custom UI components with styling - raw HTML/CSS only when required for testing
- Focus on: AI connection works, Document structure produced, Quality output
- Add styling ONLY when user specifically requests it

**The "Generation Engine" Concept:**
The system acts as middleware between input and AI:
- Receives user input (text)
- Applies structure context (if template selected)
- Applies tone settings
- Builds prompt with template context
- Sends to AI
- Parses and structures AI output into document sections
- Returns structured document

This is NOT an agent - this is the generation engine that produces documents from text input.

**Tech Stack:**
- AI: DeepSeek API (via admin-configured key)
- Real-time: Socket.io WebSocket
- Streaming: Server-sent events for chunked response

**Milestone Dependencies:** M1 (Infrastructure), M2 (Frontend Core), M3 (State Management)
