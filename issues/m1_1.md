# M1_1: Project Setup (Docker + Backend Foundation)

## Overview

**Development Approach:** Backend First
Focus on backend API development first. Minimal frontend only for testing API endpoints. Add frontend styling/UI only when needed for testing.

**Container Approach:** Docker + Docker Compose
All services containerized. MongoDB, Redis, and App in containers. Easy local development with single command.

**Milestone Dependencies:** M1 (Infrastructure)

---

## Issue 002: Set Up Nginx Reverse Proxy

### Description
Configure Nginx as a reverse proxy in front of the application. This provides load balancing, SSL termination, static file caching, rate limiting, and security headers.

### Must Have
- nginx/nginx.conf: Main configuration
- nginx/ssl/: SSL certificates directory
- Rate limiting for API and auth endpoints
- Gzip compression enabled
- Security headers (X-Frame-Options, X-Content-Type-Options)
- Static file caching (60 minutes for /_next/static/)
- HTTP to HTTPS redirect (production)
- Load balancing to upstream app:3000

### Dependencies
- Milestone: M1 (Infrastructure)
- Packages: nginx:alpine

### Files
- nginx/nginx.conf
- nginx/ssl/

### Acceptance Criteria
- [ ] Nginx starts in Docker Compose
- [ ] Routes traffic to app container
- [ ] Rate limiting active
- [ ] Security headers applied
