# M9: Voice-to-Document

## Overview

**Development Approach:** Backend First
Voice-to-document functionality using recording, transcription with Whisper, and AI formatting. NO STYLING - pure functionality.

**Development Notes:**
- No Tailwind CSS - not needed
- No custom UI components with styling - use raw HTML/CSS only when required for testing
- Focus on: Recording works, transcription works, document generates, preview shows
- Add styling ONLY when user specifically requests it

**Tech Stack:**
- Frontend: React components
- Audio: Browser MediaRecorder API
- Transcription: Self-hosted Whisper API
- AI: DeepSeek API (via M4) for formatting
- State: Zustand (from M3)

**Milestone Dependencies:** M1 (Infrastructure), M2 (Frontend Core), M3 (State Management), M4 (Generation Engine), M5 (Generate Page), M7 (Preview), M8 (Document Features)
