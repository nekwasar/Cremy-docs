# M16: SEO & Public Pages

## Overview

**Development Approach:** Frontend First
SEO-optimized public pages for marketing, including landing page, format pages, conversion tool, translate page, and more. NO STYLING - pure functionality.

**Development Notes:**
- No Tailwind CSS - not needed
- No custom UI components with styling - use raw HTML/CSS only when required for testing
- Focus on: Pages load, SEO metadata, functionality works
- Add styling ONLY when user specifically requests it

**Tech Stack:**
- Frontend: React components
- SEO: Next.js metadata API
- Analytics: M14 events

**Milestone Dependencies:** M1 (Infrastructure), M2 (Frontend Core), M11 (File Conversion)

## Issue 009: SEO Conversion Pages (Dynamic Route)

### Description
Create dynamic SEO conversion pages (handles all 200 paths with 1 file).

### Must Have
- GET /convert/[from]/[to] route
- Single dynamic file: app/convert/[from]/[to]/page.tsx
- Reads [from] and [to] params
- Generates format-specific:
  - Hero: "[From] to [To] Converter - Free Online"
  - Upload zone
  - Quality toggle: "Free" / "AI-Enhanced"
  - Features section
  - How it works (3-step)
  - Related tools (5-10 links)
  - FAQ section
  - Schema.org
- SEO metadata per page
- 404 if invalid format combination

### Format Pairs (200 total):
- Core (100): PDF ↔ DOCX, DOC, JPG, PNG, XLSX, CSV, PPTX, TXT, HTML
- Medium (100): ODT, RTF, EPUB, MOBI, ODS, XML, JSON, WEBP, TIFF, MD ↔ all above

### Dependencies
- Milestone: M11 (File Conversion)

### Files
- app/convert/[from]/[to]/page.tsx

### Acceptance Criteria
- [ ] All 200 URLs work
- [ ] SEO metadata unique per page
- [ ] 404 for invalid pairs
