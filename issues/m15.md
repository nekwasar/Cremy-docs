# M15: Payments & Subscriptions

## Overview

**Development Approach:** Backend First
Payment and subscription system with credit purchases and Pro subscriptions. Supports Flutterwave, Paystack, Stripe, and PayPal for global and local payment coverage. NO STYLING - pure functionality.

**Development Notes:**
- No Tailwind CSS - not needed
- No custom UI components with styling - use raw HTML/CSS only when required for testing
- Focus on: Payments work, credits update, subscriptions work
- Add styling ONLY when user specifically requests it

**Payment Processors:**
- **Flutterwave** - African/local card payments, mobile money, bank transfers, international cards
- **Paystack** - Nigerian/local card payments, bank transfers, USSD, international cards
- **Stripe** - International card payments (US/Europe/global)
- **PayPal** - PayPal wallet payments (international)

**Region-Based Payment Strategy:**
- African users see Flutterwave/Paystack as primary options
- International users see Stripe/PayPal as primary options
- All processors accept international cards as fallback
- Admin configures active processors per region in M14 settings

**Tech Stack:**
- Frontend: React components
- Payment: Flutterwave + Paystack + Stripe + PayPal
- Backend: MongoDB (from M1)
- State: Zustand (from M3)

**Milestone Dependencies:** M1 (Infrastructure), M2 (Frontend Core), M3 (State Management), M13 (Data Storage), M14 (Admin pricing config)

## Issue 007: Payment Webhooks

### Description
Create webhook handlers for all processors.

### Must Have
- POST /api/webhooks/flutterwave
- POST /api/webhooks/paystack
- POST /api/webhooks/stripe
- POST /api/webhooks/paypal
- Verify webhook signatures (secret hash for Flutterwave/Paystack, signing secret for Stripe, client secret for PayPal)
- Handle: payment_succeeded, payment_failed, subscription_created, subscription_updated, subscription_cancelled, subscription_expired
- Process events asynchronously via queue
- Idempotency handling (prevent duplicate event processing)
- Retry failed webhook processing 3 times

### Dependencies
