# M15: Payments & Subscriptions

## Overview

**Development Approach:** Backend First
Payment and subscription system with credit purchases and Pro subscriptions. Supports Stripe, Paddle, and PayPal. NO STYLING - pure functionality.

**Development Notes:**
- No Tailwind CSS - not needed
- No custom UI components with styling - use raw HTML/CSS only when required for testing
- Focus on: Payments work, credits update, subscriptions work
- Add styling ONLY when user specifically requests it

**Payment Processors:**
- **Paddle** - Handles global tax/compliance
- **Stripe** - Simple card payments
- **PayPal** - PayPal wallet payments

**Tech Stack:**
- Frontend: React components
- Payment: Paddle + Stripe + PayPal
- Backend: MongoDB (from M1)
- State: Zustand (from M3)

**Milestone Dependencies:** M1 (Infrastructure), M2 (Frontend Core), M3 (State Management), M13 (Data Storage), M14 (Admin pricing config)
