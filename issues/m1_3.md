# M1_3: Database & Data Models

## Overview

**Development Approach:** Backend First
All data models are API-first. No frontend components until needed. Focus on data integrity, indexing, and relationships.

**Milestone Dependencies:** M1_1 (Project Setup), M1_2 (Authentication)

This milestone covers all database models needed for the application.

---

## Issue 001: Document Model

### Description
Create Document model for storing user documents. This is the core model for the application - tracks all generated and uploaded documents.

### Must Have
- userId: Reference to User (indexed)
- title: Document title
- content: Document content (text/markdown)
- type: Document type (generated, uploaded, template)
- status: draft, processing, completed, failed
- format: pdf, docx, txt, md
- tools: Array of tools used
- inputData: Original input (JSON)
- outputData: Generated output (JSON)
- sourceDocumentId: Reference to source (for edits)
- isPublic: Boolean for public documents
- shareToken: Token for sharing
- wordCount: Word count
- pageCount: Page count (PDF)
- processingTime: Time to process (ms)
- creditsUsed: Credits consumed
- metadata: Additional metadata (JSON)
- timestamps (createdAt, updatedAt)
- Index on userId and createdAt
- Compound index for user queries

### Dependencies
- Milestone: M1_1, M1_2
- Packages: mongoose

### Files
- src/models/Document.ts

### Acceptance Criteria
- [ ] Document model created
- [ ] User relationship works
- [ ] Proper indexing for queries
