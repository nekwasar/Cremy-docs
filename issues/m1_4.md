# M1_4: Credit System Backend

## Overview

**Development Approach:** Backend First
All credit system endpoints are API-first. No frontend components until needed. Focus on transaction integrity, history tracking, and validation.

**Milestone Dependencies:** M1_1 (Project Setup), M1_2 (Authentication), M1_3 (Database & Data Models)

This milestone covers all credit system functionality - the core monetization system for the application.

---

## Issue 001: Credit Check Middleware

### Description
Create middleware to check user credits before allowing tool usage. This is the core enforcement point for credit-based access.

### Must Have
- checkCredits function: Check if user has enough credits
- deductCredits function: Deduct credits atomically
- refundCredits function: Refund credits (on failure)
- getUserCredits function: Get current credit balance
- requireCredits function: Middleware that blocks if insufficient
- Error handling for race conditions
- Atomic operations (use MongoDB transactions)
- Logging of credit operations

### Dependencies
- Milestone: M1_1, M1_2, M1_3
- Packages: mongoose (for transactions)

### Files
- src/lib/credits.ts

### Acceptance Criteria
- [ ] Check credits works
- [ ] Atomic deduction works
- [ ] Refund works
