# M1_2: Authentication System

## Overview

**Development Approach:** Backend First
All authentication endpoints are API-first. No frontend components until needed for testing. Focus on security, rate limiting, and token management.

**Authentication Methods:**
- Email/Password authentication
- JWT tokens (access + refresh)
- OAuth (Google)
- Email verification
- Password reset

**Milestone Dependencies:** M1_1 (Project Setup)

---

## Issue 001: User Model & Schema

### Description
Create User model with all authentication-related fields. This includes email, password (hashed), name, avatar, role, verification flags, OAuth IDs, and credit tracking.

### Must Have
- Email field (unique, lowercase, indexed)
- Password field (hashed with bcrypt)
- Name field
- Avatar field (optional)
- Role field (free, pro, admin)
- Email verification flags
- Google OAuth ID (optional)
- Credit tracking (credits, usedCredits)
- Timestamps (createdAt, updatedAt)
- Password hashing on save
- comparePassword method
- JWT payload methods

### Dependencies
- Milestone: M1_1
- Packages: mongoose, bcryptjs, @types/bcryptjs

### Files
- src/models/User.ts

### Acceptance Criteria
- [ ] User model created with all required fields
- [ ] Password hashing on save
- [ ] Email unique constraint
- [ ] comparePassword works
