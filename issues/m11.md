# M11: File Conversion System

## Overview

**Development Approach:** Backend First
File conversion system supporting full matrix of formats. All conversions are COMPLETELY FREE for all users. NO STYLING - pure functionality.

**Development Notes:**
- No Tailwind CSS - not needed
- No custom UI components with styling - use raw HTML/CSS only when required for testing
- Focus on: Upload works, conversion works, download works, preview shows
- Add styling ONLY when user specifically requests it

**Key Rule:** Conversions are COMPLETELY FREE for all users (no credits needed)

**Tech Stack:**
- Frontend: React components
- Conversion: AI-enhanced + libraries
- Export: Puppeteer (PDF), docx.js (DOCX), PptxGenJS (PPTX), SheetJS (XLSX)
- State: Zustand (from M3)

**Milestone Dependencies:** M1 (Infrastructure), M2 (Frontend Core), M3 (State Management), M4 (AI Generation), M7 (Preview)

---

## SEO Architecture Note

Every conversion page in this milestone is designed for enterprise-grade SEO with the following guarantees:
- **200 unique SSG pages**, each independently indexable by Google
- **Zero thin content** — each page has 300+ words of unique content tailored to that specific conversion
- **Schema.org structured data** (SoftwareApplication + WebApplication + BreadcrumbList) in JSON-LD
- **Canonical URL enforcement** with 301 redirects for variants
- **XML sitemap** with all 200 URLs, priorities, and change frequencies
- **Internal link architecture** with 7 category hubs and 5-8 contextual related links per page
- **Core Web Vitals compliant** — all pages score >90 Lighthouse Performance

### Supported Conversion Matrix (200 Pairs Total)

## Issue 019: SEO Conversion Pages Architecture (/convert/[slug])

### Description
Create SEO-friendly conversion pages targeting 200 unique conversion pairs. **Every single page must be independently indexable by Google with zero risk of thin-content penalties.** This is the critical SEO pillar of the entire domain — each page functions as a unique, valuable landing page for organic search traffic.

### SEO Architecture Strategy
- **200 SSG pages** via `generateStaticParams` — all pre-rendered at build time, no client-side rendering
- **URL pattern:** `/convert/[source]-to-[target]` (e.g., `/convert/pdf-to-word`, `/convert/docx-to-pdf`)
- **Each page has:** unique H1, unique 150+ character meta description, unique 300+ word explanatory content section, Schema.org structured data, canonical URL, breadcrumb LD+JSON
- **Zero keyword cannibalization:** each page targets a distinct search query (e.g., "convert pdf to word" vs "convert word to pdf" are different pages with different content)
- **Internal link architecture:** category hub pages (e.g., `/convert/word` with links to all Word↔X pages) + contextual cross-links between related conversions
